passthroughs

This section allows you to create passthroughs to HTML content or remote addresses/networks. Here's an example:


packetfence=http://www.packetfence.org
The above will allow 80/tcp traffic to the resolved IP address (the LHS value is arbitrary). Passthroughs can also take the form of:

test=192.168.100.10/23
which would allow full IP to all 512 destination addresses.

proxies

This section allows you to configure locally proxied content. We typically use this to proxy tools like Stinger rather than having to continually download the latest version. Ex:


tools/stinger.exe=http://download.nai.com/products/mcafee-avert/stinger.exe
The Stinger utility could then be accessed at https://pfhostname/proxies/tools/stinger.exe.
