Configuration Reference - scan

host (text)
Default: 127.0.0.1
Host the nessus server is running on. For performance reasons, we recommend running the nessus server remotely. A passthrough will be automagically created.

live_tids (text)
Default:
If a host fails a scan AND the tid is listed in live_tids the corresponding violation will be added. If the tid is not listed here the event will be logged only. This is used to test Nessus plugins before going live.

pass (text)
Default: packet
Password to log into nessus server with.

port (text)
Default: 1241
Port nessus server is running on.

registration (toggle)
Default: disabled
Options: enabled|disabled
If this option is enabled, the PF system will scan each host after registration is complete with all nessusids.

ssl (toggle)
Default: enabled
Options: enabled|disabled
enable ssl communication with the nessus server.

user (text)
Default: admin
Username to log into nessus server with.
